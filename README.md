# network-threat-detection
Title:

Design and Implementation of a Machine Learning Pipeline for Network Threat Detection

General Objective:

Develop a complete Machine Learning pipeline for network threat detection, integrating basic MLOps best practices (code versioning, experiment tracking, model saving, and automation scripts).

Project Steps
Step 1 – Problem Understanding and Domain Study

Study cybersecurity concepts and network intrusion detection. Analyze different types of attacks (DoS, Port Scan, Brute Force). Select and understand a real-world dataset (UNSW-NB15, KDD99, or CICIDS 2017).

Step 2 – ETL and Data Preparation

Ingest network data, perform data cleaning, and handle missing values and duplicates. Apply feature engineering and validate the dataset. Store the processed data in CSV files or in MongoDB.

Step 3 – Machine Learning Modeling

Split the data into training and testing sets. Implement Machine Learning models (Decision Tree, Random Forest, Isolation Forest, or K-Means). Evaluate performance using standard metrics (accuracy, precision, recall, F1-score).

Step 4 – Mini MLOps Pipeline

Manage code using Git/GitHub. Track experiments and models using MLflow. Create training and prediction scripts (train.py and predict.py). Containerization with Docker (optional).

Step 5 – Documentation and Reporting

Write a technical report including system architecture, UML design, implementation details, experimental results, and future work.
